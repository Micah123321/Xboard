## 1. 需求

### 背景
`#/nodes` 节点管理页当前只按后端返回的 `sort` 顺序展示节点，管理员需要临时按不同字段查看节点，把目标字段值高或低的节点快速放到列表顶部，同时保留默认排序入口。

### 目标
- 在节点管理表格的业务列上加入点击排序能力。
- 每个字段点击后在“置顶排序 / 置底排序 / 默认排序”三态之间循环。
- 默认排序保持现有 `sort` + `id` 顺序，不影响“编辑排序”和“置顶节点”的后端排序能力。
- 排序在筛选之后、分页之前生效，确保翻页统计与当前排序一致。

### 约束条件
```yaml
时间约束: 无
性能约束: 本地数组排序，避免新增 API 请求；保持分页前排序成本可控
兼容性约束: 不改变后端接口契约、不改 public/assets/admin 构建产物
业务约束: 操作列、勾选列不参与排序；字段排序只影响当前前端视图，不持久化
```

### 验收标准
- [ ] 节点表格业务列可点击排序，并能循环为置顶、置底、默认。
- [ ] 默认排序恢复现有后端 `sort` 顺序。
- [ ] 搜索、筛选、分页、跨分页勾选和刷新数据仍正常工作。
- [ ] `npm run build` 在 `admin-frontend` 中通过。

---

## 2. 方案

### 技术方案
在 `NodesView.vue` 内新增排序状态、字段配置和比较器。`filteredNodes` 先复用 `filterNodes()` 得到筛选结果，再根据当前排序字段和方向进行稳定排序；无排序状态时继续调用 `sortNodesByOrder()`。表格列头使用自定义 header 渲染为按钮，按钮文案保持列名，附带当前排序状态标签。

### 影响范围
```yaml
涉及模块:
  - admin-frontend: 节点管理页表格展示交互
预计变更文件: 1 个应用代码文件，2 个方案包文件，按需同步 1 个知识库模块文件
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 排序与后端持久排序语义混淆 | 中 | UI 文案使用“置顶 / 置底 / 默认”，默认态仍按后端 `sort` 排序 |
| 不同字段类型比较不一致 | 中 | 为数字、文本、状态、权限组等字段提供明确取值函数，空值稳定落后 |
| 排序后跨分页勾选状态错乱 | 低 | 保持现有 `row-key=id`、`selectedNodeIds` 和 `syncTableSelection()` 逻辑 |

### 方案取舍
```yaml
唯一方案理由: 前端本地三态排序能满足临时查看需求，不改变后端排序接口和已有持久排序语义，改动集中且可回退。
放弃的替代路径:
  - 后端接口排序参数: 需要扩展 server/manage/getNodes 契约，且当前需求是页面临时排序。
  - Element Plus 内置 sortable: 默认只有升/降/无三态，文案和“置顶/置底”语义不可控，且复杂字段仍需要自定义比较器。
回滚边界: 回退 NodesView.vue 中排序状态、列头按钮和排序样式即可恢复原行为。
```

## 4. 核心场景

### 场景: 节点列表字段临时排序
**模块**: admin-frontend  
**条件**: 管理员打开 `#/nodes`，列表已加载。  
**行为**: 点击某个业务字段列头。  
**结果**: 第一次点击按该字段“置顶”显示，第二次点击按该字段“置底”显示，第三次点击恢复默认 `sort` 顺序。

## 5. 技术决策

### node-field-sort#D001: 使用前端本地三态排序
**日期**: 2026-05-18  
**状态**: 采纳  
**背景**: 需求强调“点下就最上 最下 默认排序”，更像当前页面的临时查看交互，而不是后端持久排序。  
**选项分析**:
| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 前端本地三态排序 | 改动集中，不改接口，响应快，可保留默认后端排序 | 仅影响当前页面视图，不跨会话保存 |
| B: 后端排序参数 | 可跨分页服务端排序，适合大数据集 | 需改接口契约，当前 getNodes 已返回全量本地分页数据 |
| C: 复用 Element Plus 内置排序 | 代码较少 | 三态文案与复杂字段比较不可控 |
**决策**: 选择方案 A。  
**理由**: 当前节点页已在前端完成筛选和分页，字段排序应与该数据流保持一致；不持久化也避免与“编辑排序”混淆。  
**影响**: 仅影响 `admin-frontend/src/views/nodes/NodesView.vue` 的表格展示顺序。

## 7. 成果设计

### 设计方向
- **美学基调**: 延续当前 Apple 风格后台基线，以低装饰、清晰反馈的表头按钮呈现排序能力。
- **记忆点**: 激活列头直接显示“置顶 / 置底”，让当前排序规则不依赖图标猜测。
- **参考**: 现有节点页表格与 Element Plus 表头结构。

### 视觉要素
- **配色**: 默认灰色表头，激活态使用项目现有蓝色 `#0071e3`。
- **字体**: 复用项目当前系统字体栈，不新增远程字体。
- **布局**: 列头内联按钮，保持表格密度和列宽稳定。
- **动效**: hover / focus-visible 轻量反馈。
- **氛围**: 不新增背景或装饰层。

### 技术约束
- **可访问性**: 列头按钮提供 `aria-label` 和键盘焦点样式。
- **响应式**: 不增加固定宽度之外的大块内容，窄列只显示短排序标签。
